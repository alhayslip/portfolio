For the Proposition

Rationale for Design Decisions

Design Decisions and Rationale

  • Rationale 1: I created a new binary column, “After Neighborhood Policing,” to determine if an incident was filed when Neighborhood Policing was established in New York City. I set the date range from June 2015 to December 2019. This decision renders a score of -1; it is deceptive, as the dates I chose for this column were selected based on independent research I did on when Neighborhood Policing was implemented in New York.
  • Rationale 2: I filtered for a data frame that captures data on complaints in 2015. This decision allocates a score of -2; it is deceptive mainly because I cherry-picked a year’s worth of data to determine if there is a difference in the number of complaints by month in 2015.
  • Rationale 3: I created a list titled month_order, which contained all the months in ascending order. Additionally, I specified that the order argument was set to 'month_order', which resulted in the months being displayed in ascending order on the x-axis. This is a score of 2, as it is an earnest decision that helps the reader understand the graph better.
  • Rationale 4: I used the hue argument “After Neighborhood Policing” in sns.countplot to compare differences across ethnic groups. This generates a score of 2, this is effective visual encoding that helps the user understand the data better in barplot.
  • Rationale 5: I used plt.legend to clearly label True/False distinctions for “After Neighborhood Policing.”. This generates a score of 2, this is effective visual encoding that helps the user understand the data better in barplot.

Final Reflection

    All the decisions that I made in the design process were contingent on performing the follownig data cleaning steps. My design process involved understanding the type of question I wanted to visualize, identifying the segments of data required to capture the data I wanted to see, and deciding on the visualization I wanted to use to showcase the trends in the data effectively.

    • I replaced the numerical content in the Month When Complaint Was Received and Month was Complaint was Closed columns with the name of the month (2 was replaced with February, etc.)
    • I changed the data type for the precinct column from object to int64
    • I dropped the null records and duplicate records from the original dataframe which contained all of the allegations
    • I combined the the Month When Complaint Was Received and Year When Complaint Was Received columns into one column, allegation month and year.
    • I converted the allegation month and year column into a date time format using the pd.to_datetime argument for the allegation month and year column.
    • I wrote an np.where statement to define a binary column After Neighborhood Policing, which defined whether the record occurred before or after neighborhood policing was implemented in June 2015.

    The most challenging part of this analysis was filtering the original data frame to ensure that the column for force was created. The intention of doing this was to initiate an np.where statement to create a boolean value based on whether the column "Specific Category of Compliant" contained the phrase force. Then, I created a new data frame titled force, which filtered for all records where the Specific Category and Complaint was set to true. Additionally, I used the isin argument to filter the force data frame even further to segment for all records between 2011 to 2019. I chose those years to account for the four years before neighborhood policing and the four years after neighborhood policing was implemented.

    My conclusion that a stronger argument can be made that neigborhood policing was not effective in reducing the number of complaints filed. The visualization used in the proposition highlighted that the number of complaints filed increased after neighborhood policing was implemented. The bar plot for the number of complaints in 2015 showed that the number of complaints increased in the months preceding the implementation of neighborhood policing. There is evidence that the implementation of Neighborhood Policing reduced the number of complaints filed for excessive force by black and hispanic citizens. However, the data does not indicate that the implementation of neighborhood policing decreased the amount of complaints filed by citizens across the board.

    I define ethical analysis and visualization as understanding how to appropriately clean, filter, and visualize data in a way that effectively demonstrates trends and patterns without a hidden agenda. The bounds to distinguish acceptable and misleading decisions are very blurred, but I use the criteria of understanding what type of decision you are going to make and what the implications of that are to determine an ethical decision against a misleading decision.

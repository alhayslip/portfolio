Histogram of Organic Foods at Whole Foods By Food Processing Score

What data cleaning decisions were made?

  • 1.Write an np.where statement using the str.contains argument to create a new column: “Organic” in the food data frame
  • 2.Create a new data frame, Organic, based on whether the value in the organic column is “Organic”.
  • 3.Create a new data frame OrganicWholeFoods, which filters for all Organic products sold at Whole Foods.
  • 4.Use np.linespace to define the range of values for the numeric values on the x-axis -axis. Use sns color palette to define the range of red colors that will be used. Use a for loop to define the range of values within each bin grouping.

What observations were made?

  • The outliers for this visualization populate in bin groups three through five, which contain food processing scores between 0.15 and 0.33.
  • The local minima for this visualization occur between the following range of values: 0.13-0.20; 0.40-0.47; 0.73-0.80; 0.93-1.00
  • The distribution of this histogram is multimodal with three distinct peaks. The second peak of the data, with food processing scores between 0.60 and 0.67, is that bin, which contains the highest numerical count of records within a bin group.
  • This histogram is skewed to the left, meaning that the mean of the numerical values is less than the numerical values. The bulk of the records contain food processing scores ranging from 0.0 to 0.60.
  • I noticed that the bins with the highest frequency count of records are between 0.60 and 0.67, as well as 0.87 and 0.93 on the right-hand side of the visualization, which is interesting, as most of the data is on the left-hand side of the histogram.
Additional Observations
  • I decided to remove the KDE plot from this visualization, as I thought it would be redundant, especially with the legend for the bins of Food Processing Scores being visualized on the right-hand side.
